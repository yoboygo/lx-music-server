# Plan: Add Web Admin Panel

## Goal
Add a web admin page with Basic Auth login that lets users view/delete devices and view synced playlists with their songs.

## Architecture
- New DO handlers for list/dislike data retrieval
- New route file `src/routes/admin.ts` serving both HTML page and JSON APIs
- All endpoints protected by HTTP Basic Auth (same pattern as existing `devices.ts`)
- Single self-contained HTML page with inline CSS/JS (no build step, no static files)

## Files to Modify

### 1. `src/durable/UserSyncDO.ts` — Add 2 new DO handlers

Add `handleGetListData()` and `handleGetDislikeData()`:

```
GET /list-data  → returns { defaultList: MusicInfo[], loveList: MusicInfo[], userList: UserListInfoFull[] }
GET /dislike-data → returns dislike rules string
```

Wire them into the `fetch()` router (around line 89-101).

### 2. `src/routes/admin.ts` — New file, all admin routes

| Route | Method | Description |
|---|---|---|
| `/admin` | GET | Serve HTML page (Basic Auth required) |
| `/api/admin/devices` | GET | List devices (proxy to DO `/devices`) |
| `/api/admin/devices/:clientId` | DELETE | Delete device (proxy to DO + KV cleanup) |
| `/api/admin/list-data` | GET | Get playlist data (proxy to DO `/list-data`) |
| `/api/admin/dislike-data` | GET | Get dislike rules (proxy to DO `/dislike-data`) |

Basic Auth logic: reuse the same pattern from `devices.ts` — decode `Authorization: Basic ...`, match against `LX_USERS`.

### 3. `src/index.ts` — Mount admin routes

Add: `app.route('/', adminRoutes)`

### 4. HTML Page Design (inline in `admin.ts`)

Single-page app with two tabs:
- **Devices tab**: Table with columns: device name, type (mobile/desktop), last connection time, delete button
- **Playlists tab**: Expandable accordion — click a list name to see songs inside. Each song shows: name, singer, source, duration. Show all lists: 试听列表, 我的收藏, and any user-created lists. Also show dislike rules below.

### 5. `src/types/config.d.ts` — Add `ADMIN_HTML` env var (optional)

Not needed — HTML will be embedded as a string literal in `admin.ts`.
